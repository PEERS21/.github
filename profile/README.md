# PEERS — Peer-driven Educational Ecosystem of Resources & Services
### PEERS — это открытая платформа для размещения и объединения студенческих проектов, созданных для облегчения процесса обучения. Мы верим в силу сообщества и инновации, которые рождаются внутри «пиров» (peers).
[![Documentation](https://img.shields.io/badge/docs-MkDocs--Material-blueviolet?style=for-the-badge&logo=materialformkdocs)](https://PEERS21.github.io/)
[![License: AGPL v3](https://img.shields.io/badge/License-AGPL_v3-blue.svg?style=for-the-badge)](https://www.gnu.org/licenses/agpl-3.0)
#### 🎯 Наша философия
Платформа PEERS создана для того, чтобы делать обучение удобнее, прозрачнее и эффективнее.
* Для студентов: Мы предоставляем инструменты, которые делают обучение удобнее (планировщики, визуализаторы, агрегаторы данных).
* Академическая честность: PEERS категорически не поддерживает и не предоставляет сервисы для списывания, «читинга» или обхода проверок. Любой проект, нарушающий академическую честность, будет немедленно удален. Мы создаем удобства, а не способы обхода правил.
* Независимость: Данная платформа является частной инициативой студентов. Мы не связаны напрямую со АНО «Школа 21», ПАО «Сбербанк» или их дочерними структурами.
#### 🛡 Безопасность
Мы серьезно относимся к данным пользователей. Все сервисы обязуются:
* Использовать HMAC для защиты персональных данных.
* Не хранить секреты в переменных окружения в открытом виде (использовать Docker Secrets).
* Запрещать доступ отчисленым и не студентам Школы 21.
* Не передавать данные 3-им лицам.
* Не использовать полученные данные в личных или коммерческих целях.
#### 🏗 Общая инфраструктура
Для всех сервисов внутри сети PEERS по умолчанию доступны следующие общие [ресурсы](https://PEERS21.github.io/ "Документация").
#### 🚀 Как добавить свой сервис в организацию PEERS
1. Заявка: Создайте Issue в этом репозитории для презентации идеи и готового пилотного проекта.
2. Установление того что Вы явялетесь пиром Школы 21 и не отчислены через логин в rocket chat.
3. Репозиторий: Для вашего проекта будет создан отдельный репозиторий внутри организации github в случае одобреня.
4. Стандарты разработки:
  * Использование Docker обязательно.
  * Чувствительные данные должны считываться из файлов секретов в `/run/secrets/` (Docker Secrets).
  * Код будет лицензирован под AGPLv3.
  * Асинхронность.
  * Документация.
#### ⚖️ Лицензирование и Условия
* Лицензия: Программный код всех сервисов организации распространяется под лицензией GNU AGPLv3. Это гарантирует, что все улучшения и форки останутся открытыми. Нарушение лицензии влечёт за собой "Пенальти" за каждый день нарушения.
* Правила: Использование данных и ресурсов платформы регулируется [документом](https://github.com/PEERS21/.github/blob/main/profile/TERMS.md "Пользовательское Соглашение").
